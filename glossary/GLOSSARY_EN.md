| **Preprocessing Pipeline** | The ordered sequence of image transformation operations applied prior to CNN input, updated in v2.1 to a 5-component system: (1) FOV standardization (fundus circle detection via Hough transform, black border removal, centering, resize to 512×512), (2) Green channel imaging (extract green channel from RGB), (3) Normalization (pixel intensity scaling to [0,1]), (4) CLAHE enhancement (applied in LAB color space on L-channel, dynamic clip limit), (5) HSV contrast enhancement. ## 1. INCONSISTENCY SUMMARY

### 1.1 Feature Extraction — Dual Usage
**Term:** Feature extraction
**Inconsistency:** Used in two distinct senses: (a) the general process by which CNN layers learn spatial representations (Ch. 2, §2.2.1), and (b) a specific transfer learning strategy where a pre-trained model's convolutional base is used without further weight updates (Ch. 4, §4.4.2 — "ResNet50: Feature Extraction versus End-to-End Fine-Tuning").
**Recommendation:** Create disambiguation suffix: "Feature Extraction [CNN process]" vs. "Feature Extraction [transfer learning strategy]."

### 1.2 Dataset Size Discrepancy
**Term:** APTOS 2019 dataset
**Inconsistency:** LC-SAPAKOVA-2025-01 states "3,662 labeled training samples" (p. 81) for APTOS 2019. However, LC-SAPAKOVA-2025 and LC-Yesmukhamedov-2025-SELF report training set as 35,126 images and test set as 3,662 images — with the larger training set reflecting APTOS 2019 plus supplementary clinical data. The number 3,662 appears both as "APTOS 2019 labeled samples" and as "test set size" depending on the source.
**Recommendation:** Standardize: "APTOS 2019 contains 3,662 labeled samples. When combined with supplementary clinical data, the total training set is 35,126 images and the test set is 3,662 images." Clarify whether the 3,662 test set is APTOS-only or also supplemented.

### 1.3 Input Resolution
**Term:** Resizing
**Inconsistency:** The baseline model uses 256×256 pixels; the enhanced model uses 512×512 pixels (LC-SAPAKOVA-2025-01, pp. 81, 83). However, the Invariants (OD-3) specify "resizing to target resolution (256×256 or 512×512 pixels)" without binding to a single value. The KBTU card notes the dissertation uses "256×256 vs. 512×512 resolution" as a point of differentiation from its own prior work.
**Recommendation:** Clarify in the methodology which resolution applies to which experiment. Do not present as interchangeable.

### 1.4 Binary vs. Five-Class Classification in Baseline
**Term:** Shallow baseline CNN
**Inconsistency:** The baseline model uses sigmoid activation for binary classification (LC-SAPAKOVA-2025-01, p. 83), while the enhanced model uses softmax for five-class classification. This means the comparison is not purely preprocessing vs. no-preprocessing — it also involves a classification task change. This is flagged in Argument Map SC-1.1: "binary classification performance reported in Figs. 4–5; multiclass performance relationship requires explicit clarification."
**Recommendation:** The dissertation must explicitly address this confound in §4.2 or §5.2.

### 1.5 Preprocessing Pipeline Definition Change (v1.0 → v2.1)
**Term:** Preprocessing Pipeline (OD-3)
**Inconsistency:** The v1.0 definition (OD-3 v1.0) specified a 4-stage pipeline: (1) resizing, (2) normalization, (3) CLAHE, (4) data augmentation. The v2.1 definition (OD-3 v2.1) specifies a 5-component system: (1) FOV standardization, (2) green channel imaging, (3) normalization, (4) CLAHE in LAB space with dynamic clip limit, (5) HSV contrast enhancement. Data augmentation is reclassified as a training-time operation rather than a preprocessing component. Any references to the "four-stage pipeline" in existing corpus materials (self-publications, earlier drafts) must be updated or contextualized when writing dissertation chapters.
**Recommendation:** All dissertation chapters must reference the v2.1 5-component definition. When citing self-publications that describe the v1.0 4-stage pipeline, explicitly note that the pipeline has been refined and expanded. See also §6 Terminological Stabilization, Priority 1 item on pipeline standardization.

## 6. TERMINOLOGICAL STABILIZATION RECOMMENDATIONS

### Priority 1 (High Impact on Coherence)

1. **Standardize fine-tuning terminology.** Replace all instances of "Method 1," "Method 2," "Freeze," "Unfreeze Top Layers" with the dissertation's canonical terms: "frozen-layer fine-tuning" (Stage 1) and "progressive fine-tuning" (Stage 2), within the "two-stage fine-tuning protocol."

2. **Disambiguate "feature extraction."** Create explicit disambiguation between the CNN operation and the transfer learning strategy. Use "feature extraction layers" for the former and "feature extraction strategy" or "frozen-base strategy" for the latter.

3. **Disambiguate "generalization."** Always qualify: "training generalization" (OD-4, definition 1) vs. "cross-database generalization" (OD-4, definition 2). Never use the term unqualified.

4. **Standardize "preprocessing" terminology.** "Preprocessing pipeline" = the OD-3 v2.1 five-component sequence. "Contrast-adaptive preprocessing" = synonym for the full pipeline, used in thesis/objective. "Image enhancement" = informal description, acceptable in Chapter 1 introduction only. "CLAHE" = the specific contrast enhancement step (component 4).

5. **Resolve EHR/EMR conflict.** Standardize on "EHR" throughout. Note the terminological distinction at first use if relevant.

6. **Standardize pipeline terminology — all dissertation chapters must use the v2.1 5-component pipeline definition.** Self-publication references to the 4-stage pipeline must be contextualized as v1.0 prior work. When citing self-publications (2024–2025) that describe the original 4-stage pipeline (resizing, normalization, CLAHE, augmentation), explicitly note that the methodology has been refined to the current 5-component system (FOV standardization, green channel imaging, normalization, CLAHE in LAB, HSV enhancement) and that data augmentation has been reclassified as a training-time operation.

### Priority 2 (Moderate Impact)

7. **Clarify dataset terminology.** Always specify "APTOS 2019" for the public dataset (3,662 labeled samples) and "APTOS 2019 + supplementary clinical data" for the combined dataset (training: 35,126; test: 3,662). Never use "APTOS 2019" alone to refer to the combined dataset. Note (v3.0): EyePACS is the primary dataset (~35,126 labeled images, Kaggle labeled partition). APTOS 2019 is NOT used for robustness experiments in V3 (Experiment 3 / robustness to synthetic degradation was dropped). APTOS 2019 may appear as a secondary/supplementary dataset reference only.

8. **Standardize ResNet notation.** Use "ResNet-50" consistently (not "RESNET50" per LC-AlTimemy-2021 or "ResNet50" from v1.0).

9. **Define "convergence" formally.** Specify the criterion (e.g., validation loss plateau, early stopping trigger) used to declare model convergence.

10. **Formalize pixel normalization.** Provide the formula: x_normalized = x / 255.0, mapping pixel values from [0, 255] to [0, 1].

### Priority 3 (Lower Impact)

11. **Flag "synergy" as informal.** LC-SAPAKOVA-2025-01 uses "synergy between preprocessing and CNN architecture" without formal definition. Either define operationally or replace with precise phrasing (e.g., "combined effect of preprocessing and architectural enhancement").

12. **Standardize "robustness."** Define the specific meaning at each point of use: training stability, cross-database consistency, or deployment resilience.

13. **Address the sensitivity formula anomaly.** Create a standing footnote or methodological note regarding LC-AlTimemy-2021 Eq. 3 (Sen = TP/(TP+TN)) to be used whenever citing that source's sensitivity/accuracy figures.
