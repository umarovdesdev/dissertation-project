**Негізгі метрикалар (маңыздылығы бойынша):**

| # | Метрика | Не бағалайды | Неліктен маңызды |

## 3. Баяндаушы сөзі

Модельді бағалау үшін төрт деңгейлі метрикалар жүйесі құрылды.

**Бірінші деңгей — негізгі метрикалар.** Маңыздылығы бойынша: Weighted F1-score, ROC-AUC, Cohen kappa және Accuracy. 

F1-score — бірінші кезекте, өйткені датасеттердегі класс теңгерімсіздігін ескеру маңызды. Accuracy мұндай жағдайда жеткіліксіз сол үшін Weighted F1.

ROC-AUC пен Cohen kappa — реттелген деректер бойынша модельдің дәлдігін көрсетеді.

**Екінші деңгей — доминанттылық критерийі.** Full pipeline Baseline-нан жақсы екенін дәлелдеу үшін үш шарт бір мезгілде орындалуы тиіс: F1-ге кемінде 5 пайыздық пункт өсім, AUC-қа кемінде 2 пункт өсім, және kappa деградациясыз. Бір метриканың жоғары болуы жеткіліксіз — бұл ішінара жақсарту емес, толық доминанттылық екенін көрсетеді.

**Үшінші деңгей — клиникалық метрикалар.** Скрининг сценарийі үшін DR grade 2-ден жоғарыны referable DR деп белгілеп, sensitivity мен specificity есептеледі.

**Төртінші деңгей — сапа бақылау.** ECE мен Brier Score модельдің калибрациясын бағалайды ал Train-test gap артық оқудың индикаторы.

## 4. Қосымша

- **Weighted F1-score** — Precision мен Recall-дің гармоникалық ортасы, әр класстың салмағы оқыту жиынындағы үлесіне пропорционал. Класс теңгерімсіздігі жоғары датасеттерде macro F1-ден дәлірек.
- **ROC-AUC (macro, one-vs-rest)** — әр класс үшін ROC қисығының астындағы аймақ есептеліп, макро-орта алынады. 0.5 = кездейсоқ, 1.0 = мінсіз. Шешім шегіне (threshold) тәуелсіз.
- **Cohen κ (quadratic weights)** — екі бағалаушы (модель мен дәрігер) арасындағы келісімді өлшейді; quadratic weights реттелген шкалада алыс қателерді жақын қателерден ауыр бағалайды. DR 0→4 қатесі DR 0→1 қатесінен 16× ауыр.
- **Accuracy** — дұрыс жіктелген суреттердің жалпы үлесі. Класс теңгерімсіздігі жоғары жағдайда жеткіліксіз — сондықтан F1 бірінші кезекте.
- **Доминанттылық критерийі (EH-3)** — ΔF1 ≥ 5pp AND ΔAUC ≥ 0.02 AND Δκ ≥ 0. Үш шарт конъюнкциясы. Ішінара жақсару жеткіліксіз — толық доминанттылық талап етіледі.
- **Sensitivity (Recall)** — TP / (TP + FN). Referable DR контексінде: ауру көздердің қанша пайызы дұрыс анықталды. Жоғары sensitivity = аз false negative = аурудан жіберіп алу аз.
- **Specificity** — TN / (TN + FP). Сау көздердің қанша пайызы дұрыс жіктелді. Жоғары specificity = аз false positive = қажетсіз жіберулер аз.
- **PPV (Positive Predictive Value)** — TP / (TP + FP). Модель «referable DR» деген кезде, шынымен ауру болу ықтималдығы.
- **NPV (Negative Predictive Value)** — TN / (TN + FN). Модель «DR жоқ» деген кезде, шынымен сау болу ықтималдығы.
- **ECE (Expected Calibration Error)** — модельдің сенімділігі мен нақты дәлдігі арасындағы орташа айырмашылық. Төмен ECE = жақсы калибрация.
- **Brier Score** — ықтималдық болжамдарының орташа квадраттық қатесі. 0 = мінсіз, 1 = ең нашар.
- **Train-test gap** — train F1 мен test F1 арасындағы айырмашылық (пайыздық пунктпен). 15pp шегі — артық оқудың бақылау индикаторы: gap > 15pp болса, модель оқыту деректеріне артық бейімделген.
- **Referable DR** — DR grade ≥ 2 (орташа NPDR және одан ауыр). Маман бағалауын қажет ететін жағдайлар. Бинарлы скрининг шешімі осы шекпен анықталады.
- **Sensitivity** — ауру пациент ішінен қаншасын дұрыс таптық. Төмен болса → ауруды жіберіп аламыз (қауіпті).
- **Specificity** — сау пациент ішінен қаншасын дұрыс таптық. Төмен болса → сау адамды қажетсіз маманға жібереміз (ресурс шығыны).
