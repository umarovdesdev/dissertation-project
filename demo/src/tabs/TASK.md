## Проверка 1 — «Это фундус?»

Чистая клиентская эвристика на канвасе. ~95% точность на нормальных снимках.

Четыре независимых теста, считаем «фундус» если ≥ 3 из 4 прошли:

1. **Круглая ROI на тёмном фоне.** Бинаризация по luma > 15, bounding box не-чёрных пикселей. Условие: `w/h ≈ 1.0 ± 0.2` и доля не-чёрного в боксе > 60%.
2. **Тёмные углы (виньетка).** Средняя яркость в 4 угловых патчах (~5% от размера) < 20.
3. **Красно-оранжевая доминанта.** Внутри ROI: `mean(R) > mean(G) > mean(B)` и `R/B > 1.8`. Отсекает селфи, документы, скриншоты.
4. **Не плоско.** Стандартное отклонение luma > 20. Отсекает однотонные картинки.

Поведение при провале: **не блокировать**, показать жёлтый chip-предупреждение, разрешить override.

## Проверка 2 — правый/левый глаз

Анатомическая основа: **диск зрительного нерва (ДЗН)** всегда с **назальной** стороны.

- **Правый глаз (OD)** → ДЗН на **левой** половине снимка
- **Левый глаз (OS)** → ДЗН на **правой** половине снимка

### Алгоритм (baseline ~88–92% на EyePACS)

Сравнение средней яркости левой и правой половины внутри ROI фундуса. Половина с ДЗН ярче.

```js
function detectLaterality(imageData, roi) {
  const { x, y, w, h } = roi;
  let leftSum = 0, rightSum = 0, leftN = 0, rightN = 0;
  const mid = x + w / 2;
  const d = imageData.data;
  for (let j = y; j < y + h; j += 2) {
    for (let i = x; i < x + w; i += 2) {
      const k = (j * imageData.width + i) * 4;
      const luma = 0.299*d[k] + 0.587*d[k+1] + 0.114*d[k+2];
      if (luma < 10) continue;          // вне диска фундуса
      if (i < mid) { leftSum += luma; leftN++; }
      else        { rightSum += luma; rightN++; }
    }
  }
  const L = leftSum / Math.max(leftN, 1);
  const R = rightSum / Math.max(rightN, 1);
  const ratio = (L - R) / ((L + R) / 2); // > 0 → ДЗН слева → правый глаз
  if (Math.abs(ratio) < 0.03) return { guess: null, conf: 0 };
  return {
    guess: ratio > 0 ? 'right' : 'left',
    conf: Math.min(1, Math.abs(ratio) * 20),
  };
}
```

### Где алгоритм шумит
- Сильная катаракта / мутные среды — диск теряет яркостный контраст.
- Макроснимки периферии без ДЗН в кадре.
- Сильно децентрированные кадры.

### Улучшения (по возрастанию точности и стоимости)

- **+3–5 п.п.:** Гауссов блюр радиуса `~w/30`, argmax по luma, проверка половины ROI с пиком. ~95%.
- **+5–7 п.п.:** Tiny ONNX-модель (MobileNetV3-small, 2 класса, ~2 МБ) через `onnxruntime-web`. Для demo — overkill.

Для текущего этапа берём **baseline** (сравнение половин). Улучшение до argmax-яркости — кандидат на v2, если будут жалобы на ложные срабатывания.

## Интеграция в `EyeSlot`

```js
reader.onload = async (ev) => {
  const checks = await analyzeFundus(ev.target.result);
  // checks: { isFundus: bool, laterality: 'left'|'right'|null, confidence: number }
  onPick({ src: ev.target.result, name: file.name, fromUpload: true, checks });
};
```

UI: chip под превью рядом с бейджем `dims`. Состояние `checks` хранится рядом с `dims` в `EyeSlot`.

### Сценарии UI

| Условие | Что показываем |
|---|---|
| `!checks.isFundus` | Жёлтый chip: «Похоже, это не снимок глазного дна». Не блокировать. |
| `checks.laterality && checks.laterality !== eye && checks.confidence > 0.5` | Красный chip: «Похоже на ПРАВЫЙ/ЛЕВЫЙ глаз — слот другой». Кнопки `[Поменять местами]` и `[Оставить]`. |
| `checks.confidence < 0.5` | Ничего не показываем (модель неуверена — не мешаем пользователю). |
| Всё ок | Без chip, как сейчас. |

Свап слотов реализуется в `Demo` через перестановку `leftImg`/`rightImg` в state.
