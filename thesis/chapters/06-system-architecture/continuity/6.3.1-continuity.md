# CONTINUITY NOTE — §6.3.1 Telemedicine and Portable Device Support for Rural Deployment

**Version:** 6.0.0 · Carry into §6.3.2. Carry-in was 6.2.2-continuity (local grade-producing path as the deployed unit; three-sub-section map).

---

**Key concepts established:** the §6.2 local grade-producing path deployed in three settings — (6.3.1.1) distributed telemedicine via the Ingestion module + store-and-forward across a peripheral-site/reading-centre split, with FR-7 error-handling surfacing unscreenable captures locally; (6.3.1.2) national-eHealth integration through the §6.1.2 FHIR/HL7 PACS/EHR layer, with the ER-model entities (Patient, Telemedicine Session, Device, Diagnostic Result, AI Model) as the persisted unit of exchange and infrastructure as an inherited prerequisite (DGL-4); (6.3.1.3) real-time remote screening as a **design feasibility, not a demonstrated capability** (SB-4.1/SB-4.3), with portable/non-mydriatic capture handled by the Preprocessing Engine's normalization stages. The unifying property: local grade-producing path + asynchronous hospital exchange = admissible distributed/low-resource deployment by design.

**Terms introduced (framing):** peripheral capture site vs reading centre; telemedicine session (as ER entity + unit of exchange); non-mydriatic/portable fundus camera; task-shifting (from #45, as cited precedent). Carried, already defined: store-and-forward, PACS/EHR, FHIR/HL7 (§6.1.2); Ingestion/Preprocessing/Inference modules (§6.2); OD-6 envelope (§6.1.1).

**Argument thread:** the feasibility argument is grounded in the *architecture's* envelope-fit (store-and-forward + host-budgeting), NOT in impact projections. External deployment precedent (#14 primary-care meta-analysis with prevalence-PPV + no-general-recommendation caveats; #45 prospective LMIC national programme — deployment feasibility only, specialist comparison NOT echoed; #34 35–55% compliance gap) contextualizes the modality but is never imported as this system's performance (SIR-3/SIR-2). Kazakhstan benefit figures held as third-party projections (SIR-8). Active claim at section end: distributed/portable/low-resource deployment is admissible by design; field performance reserved.

**Final topic:** FIG-6.1 dashboard as the reading-end interface illustration (not results); explicit hand-off to §6.3.2 (physician-in-the-loop decision-support surface).
