**Key concepts established:**
- **Convolution**: learnable filters over local receptive fields with **weight sharing** → translation-equivariant **feature maps**; parameter efficiency vs dense connection (AlexNet #65 established large-scale CNN viability).
- **Nonlinearity (ReLU)** and **pooling** (max-pooling, downsampling → local invariance + receptive-field growth); the conv→nonlinearity→pooling block stacked into a **feature hierarchy** (low-level edges/textures → abstract patterns; FIG-2.2).
- **Depth → trainability problem (degradation)**, resolved by **residual learning** (#66 He, identity shortcuts; ResNet-50 = dissertation backbone) and **dense connectivity** (#67 Huang, feature reuse, parameter efficiency) — two connectivity mechanisms, not ranked.
- **Fundus tension**: pooling discards small-feature resolution; even attention-augmented, pooling-limited architectures (#08 EAD-Net) nearly fail on microaneurysms → architecture alone does not solve small-low-contrast-lesion detection → **model = preprocessing + CNN** (CENTRAL_THESIS), and CFC-2.8 (no isolated-factor attribution) pre-emptively honored.
- SB-3.1: backbones are established choices, **not** claimed optimal.

**Terms introduced (now available without re-definition):** convolution, learnable filter, local receptive field, weight sharing, feature map, translation equivariance, ReLU, max-pooling, downsampling, feature hierarchy, residual learning / identity shortcut, ResNet-50, dense connectivity / DenseNet, feature reuse, parameter efficiency, dilated convolution, attention (named via #08).
