### CONTENT SPECIFICATION

**Section objective:** Specify, stage by stage and in execution order, the canonical 8-stage V5 preprocessing pipeline (OD-3) as the engineering realization of the P2 paradigm — defining for each stage its transformation, its always-on/train-only status, and its design rationale — so that the pipeline is reproducible and so that later sections (Exp 1/2) can treat it as a precisely defined independent variable; and to ground the two stages backed by real validation artifacts (Stage 1 OD/fovea detection — FIG-3.10/RES-VAL; Stage 7 dataset-specific normalization — RES-NORM) in those artifacts with honest reporting of detector reliability.

**Argumentative spine (required):**
- **Thesis of this section:** The diagnostic model is a two-stage system, `model = preprocessing + CNN`; the preprocessing stage is a deterministic, order-dependent, eight-stage transformation (OD-3) that is specified here as an integral model component — the engineering realization of the P2 commitment — rather than as ancillary data preparation.
- **Reasoning chain:**
  1. Frame: under P2 the feature space the CNN sees is co-determined by preprocessing; therefore the pipeline must be specified with the same rigor as the network (contrast with P1 practice, CFC-2.9).
  2. Specify the eight stages in order (0→7) per OD-3, each with transformation, status, and rationale; identify which are always-on and which is train-only (Stage 6).
  3. Ground the empirically-validated stages: Stage 1 rests on classical OD/fovea detection whose reliability was measured on IDRiD (RES-VAL/FIG-3.10) — report honestly that OD localization is moderate and fovea localization is unreliable, which is *why* the design uses a low-confidence fallback and an adaptive augmentation σ; Stage 7 uses dataset-specific statistics actually computed on EyePACS (RES-NORM).
  4. Define the pipeline's ACTIVE (full V5, 4-channel) vs ABSENT (baseline, 3-channel) states (OD-3) — the operational construct that Exp 1 contrasts — and name the 7-level ablation decomposition (Exp 2) without evaluating it.
- **Conclusion / hand-off:** The pipeline is fully specified as the H-1 independent variable; §3.1.2 deepens Stage 5 (dual-constraint CLAHE), §3.1.3 Stage 6 (augmentation), §3.1.4 the external-ingestion protocol that precedes Stage 0 for clinical data.

**Required content elements:**
1. Paragraph-1 paradigmatic preamble: `model = preprocessing + CNN`; P2 conceptual decision vs P1 practice (CFC-2.9 permitted phrasings; cite gulshan-2016.md §15/§18). Connect to §2.C hand-off (Chapter 2 foundations → Chapter 3 methodology).
2. Stage 0 — Canonical flip (L→R, OD right/macula left); always on. [FIG-3.2]
3. Stage 1 — OD-fovea rotation normalization (classical CV: OD brightest-region, fovea darkest-with-distance-prior; rotate OD→fovea axis horizontal; fallback skip on low confidence, fallback σ=13.0°; augmentation σ adaptive from detection uncertainty); always on. **Ground in RES-VAL/FIG-3.10 with honest reliability reporting.** [FIG-3.3, FIG-3.10]
4. Stage 2 — FOV crop + isotropic resize to 512×512 + centered zero-padding (foreground detection, not Hough); always on. [FIG-3.4]
5. Stage 3 — FOV mask generation (binary 1.0/0.0 → 4th channel); always on. The explicit-mask-as-stage is a named design feature (RESEARCH_ARCHITECTURE §10). [FIG-3.5]
6. Stage 4 — Flat-field correction (corrected = image − GaussianBlur(image, σ) + 128; σ = 0.07·D, D = FOV diameter from mask; inside mask only); always on. [FIG-3.6]
7. Stage 5 — CLAHE (dual-constraint clip limit on LAB L-channel; tile grid 8×8; stochastic p=0.8 train / deterministic inference). State the rule, defer full formalization to §3.1.2; DGL-5 portability caveat. [FIG-3.7]
8. Stage 6 — Augmentation (train only; unified affine [rotation σ from Stage 1, zoom 0.9–1.1, shear/stretch] + brightness/contrast + PCA color jitter); defer detail to §3.1.3. [FIG-3.8]
9. Stage 7 — Dataset-specific normalization (ToTensor HWC→CHW [0,1]; channel-wise (x−mean)/std from EyePACS training set after Stages 0–4, mask=1.0 pixels only; output 4×512×512). **Ground in RES-NORM (real EyePACS stats).** [FIG-3.9]
10. ACTIVE vs ABSENT definition (OD-3): full V5 (4-ch) vs baseline (stretch-resize + ImageNet normalize, 3-ch, no mask). Name the construct as the Exp-1 contrast; name the 7-level Exp-2 ablation ladder (NC-17 caveat). [FIG-3.1, FIG-3.14]
11. Closing hand-off to §3.1.2/§3.1.3/§3.1.4.
