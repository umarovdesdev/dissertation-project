### CONTENT SPECIFICATION

**Section objective:** Specify the documented hardware configuration and the reproducibility protocol (standardized training configuration, fixed seed/deterministic execution, fixed augmentation + schedule, versioned code), render TAB-3.1 inline, and bound all computational-efficiency statements to the documented hardware (DGL-2) while acknowledging the private-data reproducibility limit (SB-2.2).

**Argumentative spine (required):**
- **Thesis:** Reproducibility in this work is a designed property, not an aspiration: a single standardized training configuration, deterministic execution under a fixed seed, fixed augmentation and learning-rate schedules, and versioned code together make a measured difference between configurations attributable to the configuration rather than to run-to-run variation — within the bound, fixed by the documented hardware, that no efficiency claim transfers to a different compute context (DGL-2).
- **Reasoning chain:**
  1. Open from §4.1.2: the data and partitioning are fixed; the remaining setup element is the execution environment under which fitting and evaluation are performed.
  2. State the documented hardware: a single consumer-grade GPU (NVIDIA RTX 3060, 12 GB VRAM) under WSL2 with a fixed conda environment; argue analytically that this constraint is itself a design parameter — e.g. the batch size of 16 at 512×512 is a consequence of the VRAM bound, and the mixed-precision split (enabled for ResNet-50, disabled for EfficientNet to avoid fp16 overflow) is a hardware-driven engineering choice.
  3. Render TAB-3.1 inline (optimizer, batch, epochs/early-stop, loss, resolution, channels, mixed-precision split, CV, seed) from RESEARCH_ARCHITECTURE §4.0.
  4. Specify the reproducibility controls: fixed seed 42 + deterministic execution, fixed augmentation parameters, fixed LR schedule, documented software stack, versioned preprocessing/source code (App A); state what these controls buy (configuration-attributable differences).
  5. Bound it: DGL-2 — efficiency/real-time statements hold only for this hardware; SB-2.2 — results dependent on the private Clinical set carry a structural reproducibility limit; the constrained single-GPU setting instantiates the resource-limited framing (OD-6) but no real-time deployment capability is demonstrated here.
